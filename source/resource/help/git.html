Git対応について

メニューから項目を選択するかショートカットキーを押すことで、ZpDIC上でGitコマンドを実行することができます。

この機能には、GitのJava実装であるJGitを利用しています。 そのため、通常のGitがコンピュータにインストールされている必要はありません。

各コマンドの詳細

Git init

辞書データが置かれているディレクトリに新たにGitリポジトリを作成します。 該当ディレクトリがすでにあるリポジトリの管理下にある場合でも、該当リポジトリ直下に.gitフォルダがある場合を除き、新たなリポジトリを作成します。

辞書データが置かれているディレクトリで以下のコマンドを実行するのと等価です。 

git init

Git add/commit

現在開いている辞書ファイルのみをステージングし、コミットします。 同じリポジトリ内の他のファイルはステージングされません。

リポジトリの.gitフォルダが置かれているディレクトリで以下のコマンドを実行するのと等価です。 

git add (辞書ファイルの相対パス)
git commit -m "(入力されたメッセージ)"

Git checkout

現在開いている辞書ファイルのみを直前のコミットの内容に戻し、辞書ファイルを再読み込みします。 同じリポジトリ内の他のファイルは変更されません。

リポジトリの.gitフォルダが置かれているディレクトリで以下のコマンドを実行するのと等価です。 

git checkout (辞書ファイルの相対パス)

Git push

現在開いている辞書ファイルが管理されているリポジトリの特定のブランチなどをリモートリポジトリにプッシュします。 リモートリポジトリ名とブランチ名など (RefSpec) を指定できます。 リポジトリ名もブランチ名を指定しないこともでき、この場合の挙動はGitの仕様に従います。 また、https通信でプッシュする場合は、ユーザー名とパスワードの情報を入力する必要があります。

リポジトリの.gitフォルダが置かれているディレクトリで以下のコマンドを実行するのと等価です。 

git push (入力したリポジトリ名) (入力したRefSpec)
